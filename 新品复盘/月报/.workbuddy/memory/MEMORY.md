# MEMORY.md - 新品月报项目

## 项目概述
- 用户在 `C:\Users\Administrator\Desktop\新品复盘\月报\` 目录下处理新品月报数据
- 关键文件：`新品月报数据.xlsx`（156个SKU）、`process_data.py`（数据处理脚本）
- 参考格式：`C:\Users\Administrator\Desktop\新品复盘\新品数据处理报告_3.19-3.25.html`

## 数据记录规则（重要）
- **按月上架新品分别统计**，不做1-3月累计
- 1月新品：53个SKU
- 2月新品：36个SKU  
- 3月新品：67个SKU

## 低占比新品筛选标准
- 自身市占比 < 75% AND 对手有出单（competitor > 0）
- 1月低占比：24个SKU
- 2月低占比：13个SKU
- 3月低占比：35个SKU
- **总计：72个低占比新品**

## 输出文件
- `新品月报_2026年3月_增强版.html` - **最新最完整版本**，包含所有Excel字段
  - 品线/分析人/拓展类型：三月分列 + 销售额 + 均价 + 分析频次 + 市占比
  - 全量明细：销售编号、上架日期、首次出单、1/2/3月分列；支持批次+品类+分析人三重过滤
  - 未出单市场：三月对比
  - KPI卡片：9项指标
- `新品月报_2026年3月_完整版.html` - 次新版，有三月对比图表和低占比下拉
- `新品月报_2026年3月_新版.html` - 简化版，同样有下拉列表功能

## 批次判断方法（重要）
- **用最新上架日期的月份**来判断批次，而非销售编号范围
- `pd.to_datetime(df['最新上架日期']).dt.month` 映射到 1月/2月/3月批次

## 重要提醒
- 生成HTML后需验证数据完整性：全量明细应为156条，低占比应为72条
- 全量明细数据从Excel读取，而非从low_share复制
- 历史数据放在temp_data.json中

## 月报计算核心规则（重要！）
- **每月销售额 = 当月批次产品的当月销售额**，不是1-N月累计！
  - 3月销售额 = 3月批次（最新上架日期在3月）的3月销售额
  - 1月销售额 = 1月批次产品的1月销售额
  - **之前错误：用了 `edf`（所有批次合计）而非 `mdf`（当月批次），导致累加了1-3月所有批次的销售额**
- **货币单位：USD（美元），源数据就是美元，不做任何换算！**

## 常见问题处理
- 数据类型转换错误：用 `pd.to_numeric(errors='coerce')` 处理混合数据类型
- 比较操作符错误：确保比较前数据类型一致
- **Node.js v24.14.0 bug**：无法解析超过约200字符的单行JavaScript代码，Chart.js的options参数必须拆分成多行

## HTML图表代码规范
- mkBar/mkLine/mkPie函数中的`new Chart()`调用必须使用多行格式
- 不能写成单行，否则Node.js v24.14.0会报`Unexpected token ')'`语法错误
