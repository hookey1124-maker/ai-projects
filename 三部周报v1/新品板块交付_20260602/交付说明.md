## 二、做了哪些判断

### 架构判断：React纯展示层
- **决策**：砍掉所有Excel解析函数（原adapter 1732行→210行），React不做任何数据处理
- **理由**：Python管线已做全部聚合/百分比计算/ACOAS去重，React重复处理导致编码错误和16+处bug
- **数据流**：`新品检查周源数据和PLP数据.xlsx` → Python gen_html → HTML(内嵌53个JS数据块) → JSON提取(corrected_data.json) → Adapter → Page渲染

### 百分比×100三分法
- **汇总表**(share4w/acos4w/cvr等)：Python端已×100输出百分数 → React不透传×100
- **PLP明细表**(acos/acoas/cvr/ctr)：Python端保持0-1小数 → React端渲染时×100
- **格式化字符串**(汇总表cpc/cpa/acos等)：已是"$1.04"/"11.31%" → React端直接透传

### 图表库选择：Recharts替代Chart.js
- 理由：React生态原生支持，ComposedChart原生双Y轴，声明式API减少DOM操作
- 代价：不再依赖Chart.js CDN，离线可用

### 数据结构方向适配
- `priceOverview.byAnalyst` 是 `[{analyst, "$0-10": 0}]`（按分析人行）→ 渲染时行列转置
- 所有share4w/catShare4w/anShare4w时间序为 `[4.30, 5.7, 5.14, 5.21]`
- cum43Stats内部为逆时间序 `[5.21, 5.14, 5.7, 4.30]`

### 出单分布恢复HTML的4段设计
- 旧React错误：Y/N/未出单 3段（按8日出单维度）
- 修正为：有对手已出单/有对手未出单/无对手已出单/无对手未出单 4段（有/无对手 × 已/未出单）

### 四三累计与HTML完全对齐
- 移除React自增的SKU下钻（HTML没有）
- 移除React自增的运营判断/PLP列（HTML没有）
- 补回HTML有的首次出单列
- 补回7个筛选器（HTML有7个，React之前只有4个）

## 四、预期的汇报输出

### 正常情况
周报文案自动生成，包含：
```
【新品周报 5.21-5.27】
在跟SKU 140个，本周销量231（环比+24.9%），销售额$24,943（环比+41.6%）。
有对手SKU 53个，出单率90.6%。
分析及时率65.7%，超7日未分析18个。
低占比新品50个。

主要发现：
1. 出单率 90.6%，有对手未出单 5 个SKU需关注
2. 分析及时率 65.7%，超7日未分析 18 个
3. 低占比新品 50 个，竞争压力显著

下周重点动作：
1. 集中处理超7日未分析的低占比产品
2. 对有对手但未出单的SKU做竞品对标分析
3. 持续关注新品爬坡与广告ROAS表现
```

### 异常预警规则（newProductStatusRules.ts）
- **低占比高风险**：hasCompetitorUnsold.total > 0 → Severity: high
- **新品未出单**：noSaleCount > 0 → Severity: medium
- **爬坡缓慢**：8日外出单占比 > 50% → Severity: low
- **分析延迟**：总及时率 < 60% 或某分析人 < 30% → Severity: medium
- **竞品压力**：8日内出单率下降 > 10pp → Severity: low
