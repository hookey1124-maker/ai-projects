# 总销售数据业务口径冻结说明

本文档冻结当前“总销售数据”模块的核心业务口径。除特别说明外，当前页面 KPI、趋势图、排行榜、明细表和汇报文本均基于同一份 Excel 解析结果和 `src/utils/metrics.ts` 中的聚合逻辑。

## 一、周期口径

### 当前周期

- 当前周期由页面顶部“汇报周期”选择器确定。
- 默认加载数据后选中 Excel 中识别到的最后一个周周期。
- 周周期来自表头中的周销量字段，例如 `4/16-4/22销量`。

### 对比周期

- 对比周期为当前周期的前一个周周期。
- 如果当前周期是第一期，则没有对比周期，环比类指标显示为 `--` 或无变化状态。

### 近 8 周趋势范围

- 图表默认显示以当前周期为终点，向前最多 8 个周周期。
- 当可用周期不足 8 周时，展示全部可用周期。
- 后续摘要如使用趋势判断，应尽量使用同一窗口，避免图表与文字基于不同时间范围。

### 辅助字段匹配规则

- 每个周周期先识别周销量字段和同周期销售额字段。
- 市占比、直接对手出单、在售价等辅助字段按周期结束日匹配。
- 匹配候选日期为：结束日、结束日 +1、结束日 +2、结束日 +3。
- 市占比和直接对手出单使用 `M/D` 格式匹配，例如 `4/23市占比`、`4/23直接对手出单`。
- 在售价使用 `YYYY/M/D` 格式匹配，例如 `2026/4/23在售价`。
- 如果同一周期的辅助字段匹配到不同日期，会产生 warning，提醒辅助字段日期不一致。

## 二、销量口径

### 当前周期销量字段

- 当前周期销量取当前周期对应的 `xxx销量` 字段。
- 例如当前周期为 `4/16-4/22`，销量字段为 `4/16-4/22销量`。
- 空值、错误值和无法解析为数字的值按 0 参与计算。

### 维度汇总方式

- 总盘销量为当前周期所有行销量求和。
- 维度销量按产品分类、产品等级、分析人等字段分组后求和。
- `未分组`、`无在售` 等隐藏维度不进入维度表展示。

## 三、销售额口径

### 当前周期销售额字段

- 当前周期销售额取当前周期对应的 `xxx销售额` 字段。
- 例如当前周期为 `4/16-4/22`，销售额字段为 `4/16-4/22销售额`。
- 维度销售额按分组后求和。

### 销售额列缺失 warning

- 如果某个周期缺失对应销售额列，不能只静默按 0 解释业务结果。
- 当前实现会增加 warning：
  - `周期 xxx 缺失销售额列，销售额、出单均价和销售额趋势判断需谨慎。`
- 该 warning 不改变现有计算结果，只提醒销售额、出单均价和销售额趋势存在误判风险。

## 四、市占口径

### 冻结公式

当前采用汇总口径：

```text
市占比 = 汇总销量 / (汇总销量 + 汇总直接对手出单)
```

### 明确不采用的口径

- 不直接平均 Excel 行级市占比。
- 不用 SKU 行级市占比简单平均作为总盘或维度市占。

### 直接对手出单缺失规则

- 如果整个周期没有匹配到直接对手出单字段，市占比为 `null`，页面显示为 `--`。
- 如果直接对手出单字段存在，当前计算会将单元格空值、错误值和无法解析为数字的值按 0 汇总。
- 由于空值按 0 可能抬高市占判断，解析阶段会对空值比例较高的周期增加 warning。

### 直接对手出单空值 warning

当前最小 warning 规则：

- 仅检查当前周期销量大于 0 的 SKU 行。
- 如果该周期 `competitorColumn` 存在，并且这些动销行中的直接对手出单空值数量不少于 5 行，且空值比例不低于 20%，则增加 warning：
  - `周期 xxx 直接对手出单字段存在较多空值，市占判断需谨慎。`

该 warning 不改变现有市占计算结果，只提醒市占可能因对手出单空值被高估。

## 五、在售均价口径

- 当前在售均价为有效在售价的简单平均。
- 有效在售价来自周期匹配到的 `YYYY/M/D在售价` 字段。
- 会过滤空值、文本、错误值、0、负数和异常大值。
- 当前异常大值阈值为 `>= 10000`。
- 当前不按销量加权，也不按销售额加权。

## 六、出单均价口径

```text
出单均价 = 销售额 / 销量
```

- 出单均价基于当前汇总后的销售额和销量计算。
- 销量为 0 时，出单均价为 `null`，页面显示为 `--`。

## 七、趋势窗口口径

- 图表默认显示以当前周期为终点的近 8 周。
- 总盘趋势和细分趋势使用同一窗口规则。
- 摘要如果使用趋势判断，也应尽量使用同一近 8 周窗口，避免与图表表达不一致。

## 八、当前待确认项

- DataCenter 第二阶段新增的 `SalesFact.marketShare` 是 SKU 行级事实口径，便于后续统一事实表查询；当前页面展示口径仍以 `metrics.ts` 的汇总口径为准。
- DataCenter 第二阶段暂不接管 `PeriodSelector`，当前页面仍使用 `selectedPeriodIndex`。
- DataCenter 第二阶段暂不接管 KPI、趋势图、排行榜、明细表和周报文本计算，当前页面仍使用 `metrics.ts`。
- 后续如要让 DataCenter 驱动页面，需要先确认事实表聚合后的结果与当前 `metrics.ts` 输出完全一致。
- 广告、价格、新品等数据源当前只保留 registry 状态，不接入真实页面和计算逻辑。
