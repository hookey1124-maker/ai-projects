# UI 风格指南

## 整体风格

当前系统是业务分析看板风格：浅灰背景、白色卡片、深色侧边栏、蓝色作为主强调色。页面应保持克制、清晰、适合反复查看和横向对比。

## 卡片样式

卡片使用白色背景、浅色边框、8px 圆角和轻阴影。不要使用大面积装饰渐变，也不要把多个卡片继续套进更大的装饰卡片。

推荐复用：

- `dashboard-section`
- `table-card`
- `summary-panel`
- `module-section`

## 标题层级

页面顶部使用模块标题。模块内部标题层级由模块负责人按内容复杂度自行设计，避免为了占位而预设固定区域。

## KPI 卡片

需要 KPI 时可使用网格布局，主指标大号加粗，指标名使用 muted 色。新增模块是否展示 KPI、展示哪些 KPI，由模块负责人按业务目标设计。

## 状态标签

状态标签使用胶囊样式：

- 低风险：绿色
- 中风险：琥珀色
- 高风险：红色
- 未知或待接入：灰色

统一组件为 `components/common/StatusTag.tsx`。

## 表格样式

表格应保留当前看板风格：表头清晰、数字右对齐、异常标签醒目、可排序字段要有明确交互反馈。不要在模块中重新定义全局表格样式。

## 筛选器样式

需要筛选器时使用白色卡片感容器、浅色边框、紧凑间距。是否需要筛选器以及筛选维度由模块负责人设计。

## 空数据样式

空数据使用统一 `EmptyState`，文案说明当前模块等待接入的数据源或规则。空状态不应被写成错误状态，除非数据源解析失败。

## 汇报文本样式

需要汇报文本时使用白底、清晰分段和可复制按钮。文字应面向周报复盘，避免直接暴露技术字段名。是否展示汇报输出区域由模块负责人设计。
