# 销量预估模块开发任务

## 模块目标

开发销量预估模块，基于历史销量、趋势和业务假设输出销量预测相关结论。

## 只允许修改的目录

`src/modules/salesForecast/`

## 禁止修改的文件

禁止修改 DataCenter 核心文件、Sidebar、周期系统、全局 CSS、总销售模块和其他模块目录。

## 数据周期口径

`FORECAST_WEEK`，预测周。预测周期不能和销售周、广告自然周或价格快照直接混做环比。

## 页面设计要求

模块负责人自行设计页面内容、指标展示、图表表格、异常规则和汇报输出方式。页面必须保留模块标题、简介、数据周期、周期口径、数据来源状态和空数据状态。

## Adapter 要处理什么数据

通过 `salesForecastAdapter.ts` 接入历史销量、预测表或模型输出。具体输入输出字段由模块负责人自行设计并声明类型。

## Rules 要判断什么异常

异常方向、阈值和严重程度由模块负责人根据销量预估业务目标自行设计。预测周期比较必须遵守相同周期类型规则。

## Summary 要输出什么汇报文字

汇报结构和文案由模块负责人自行设计，要求面向业务复盘，能说明主要发现和建议动作。

## 验收标准

只改模块目录；不直接解析 Excel；展示周期、口径、数据来源状态；保留空数据状态；`npm run build` 通过。
