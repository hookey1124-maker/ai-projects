# 新品状态模块开发任务

## 模块目标

开发新品状态模块，跟踪新品上市、动销、爬坡和异常状态，输出可用于周报的新品结论和下周动作。

## 只允许修改的目录

`src/modules/newProductStatus/`

## 禁止修改的文件

禁止修改 DataCenter 核心文件、Sidebar、周期系统、全局 CSS、总销售模块和其他模块目录。

## 数据周期口径

`SALES_THU_TO_WED`，周四至周三。只能和同周期类型的新品上期数据做环比。

## 页面设计要求

模块负责人自行设计页面内容、指标展示、图表表格、异常规则和汇报输出方式。页面必须保留模块标题、简介、数据周期、周期口径、数据来源状态和空数据状态。

## Adapter 要处理什么数据

通过 `newProductStatusAdapter.ts` 接入新品清单、新品上市日期、新品周度表现，并可与 DataCenter 的 `BaseSkuRecord` 合并。具体输入输出字段由模块负责人自行设计并声明类型。

## Rules 要判断什么异常

异常方向、阈值和严重程度由模块负责人根据新品业务目标自行设计，但必须只在相同周期类型内做环比。

## Summary 要输出什么汇报文字

汇报结构和文案由模块负责人自行设计，要求面向业务复盘，能说明主要发现和建议动作。

## 验收标准

只改模块目录；不直接解析 Excel；页面展示周期、口径、数据来源状态；保留空数据状态；`npm run build` 通过。
