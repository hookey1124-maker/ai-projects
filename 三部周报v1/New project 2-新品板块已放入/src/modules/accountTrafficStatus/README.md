# 账号流量状态模块

## 模块目标

跟踪账号流量、访问和转化表现，为周报输出账号流量相关结论和动作建议。

## 当前开发状态

当前为干净开发壳子，仅保留模块标题、简介、数据周期、周期口径、数据来源状态和空状态提示。

## 允许修改文件

仅修改 `src/modules/accountTrafficStatus/` 目录内文件。

## 禁止修改文件

禁止修改 DataCenter 核心文件、Sidebar、周期系统、全局 CSS、总销售模块和其他模块目录。

## 数据来源

账号流量数据必须通过 `accountTrafficStatusAdapter.ts` 接入。页面不能直接解析 Excel。

## 周期口径

`NATURAL_WEEK_MON_TO_SUN`，自然周，周一至周日。账号流量模块不能使用销售周期做环比。

## 页面设计

模块负责人自行设计页面内容、指标展示、图表表格、异常规则和汇报输出方式。必须保留主项目框架中的模块标题、简介、数据周期、周期口径、数据来源状态和空数据状态。

## Adapter 输入输出

Adapter 负责把账号流量报告或店铺访问报告转换为模块内部数据结构，输出类型由模块负责人定义并在代码中明确声明。

## Rules 规则要求

Rules 负责账号流量异常判断，规则方向和阈值由模块负责人根据业务目标设计。

## Summary 文案要求

Summary 负责账号流量周报文案，输出结构和内容由模块负责人设计，但必须面向业务汇报。

## 验收标准

- 不修改模块外文件。
- 页面展示数据周期、周期口径、数据来源状态。
- 不直接解析 Excel。
- `npm run build` 通过。
