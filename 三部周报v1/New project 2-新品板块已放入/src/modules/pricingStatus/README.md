# 价格状态模块

## 模块目标

跟踪价格变化、价格风险及其对业务表现的影响，为周报输出价格相关结论和动作建议。

## 当前开发状态

当前为干净开发壳子，仅保留模块标题、简介、数据周期、周期口径、数据来源状态和空状态提示。

## 允许修改文件

仅修改 `src/modules/pricingStatus/` 目录内文件。

## 禁止修改文件

禁止修改 DataCenter 核心文件、Sidebar、周期系统、全局 CSS、总销售模块和其他模块目录。

## 数据来源

价格数据必须通过 `pricingStatusAdapter.ts` 接入，可复用 DataCenter 的价格和销量基础字段。页面不能直接解析 Excel。

## 周期口径

`DATE_SNAPSHOT`，日期快照。价格快照不能和广告自然周或销售周直接混做环比。

## 页面设计

模块负责人自行设计页面内容、指标展示、图表表格、异常规则和汇报输出方式。必须保留主项目框架中的模块标题、简介、数据周期、周期口径、数据来源状态和空数据状态。

## Adapter 输入输出

Adapter 负责把价格快照、调价表或价格监控数据转换为模块内部数据结构，输出类型由模块负责人定义并在代码中明确声明。

## Rules 规则要求

Rules 负责价格异常判断，规则方向和阈值由模块负责人根据业务目标设计。

## Summary 文案要求

Summary 负责价格周报文案，输出结构和内容由模块负责人设计，但必须面向业务汇报。

## 验收标准

- 不修改模块外文件。
- 页面展示数据周期、周期口径、数据来源状态。
- 不直接解析 Excel。
- `npm run build` 通过。
