## V1（✅ 已完成）：Prompt 驱动生图 + 文案先行

```
market_intel.json + listing_output.json
  → prompt_builder.py（DeepSeek 生成 6 张套图 Prompt）
    → batch_image_gen.py（maiziAI 生图）
      → image_validator.py（GLM-4V 验图）
        → 导出 .docx + .xlsx → SKU 目录
```

**交付物**：
- 生图/ 模块（prompt_builder / image_gen / batch_image_gen）
- 视觉模块/verification/（image_validator / listing_validator / schema_validator）
- 爬虫/ 模块（product_intel / batch_xlsx / summary_xlsx）
- 6 张套图类型（main / fitment / install / selling_points / detail / shipping）
- md2docx.py Markdown → Word 转换

## V2（✅ 已完成）：多源融合规范化引擎

每个字段从 compatibility API、Item Specifics、标题、图片 四个来源提取 → 按字段优先级链融合 → 冲突检测 → 置信度评分。

**交付物**：
- `parser/` — 10 字段解析器（vehicle / cab / position / count / finish / material / color / structure / bed_length / accessory），含 Front-Rear/Both-Sides 多位置套件 + Inner/Outer/Upper/Lower 方位 + Exterior 防误判、Black-Chrome/Satin/Anodized 等 14 种表面处理 + Chrome 滞后匹配、N-piece 结构检测 + assemble 修复、Zinc Alloy/Nylon/PP 等 20 种材质
- `engine/` — 融合引擎（resolver / conflict / confidence / canonicalize），canonicalize 含 fitment_type from_title() 正则
- `config/` — field_priority.json / field_risk_weight.json / canonical_maps.json（90+ 映射）/ noise_keys.json

## V2.5（✅ 已完成）：竞品自动发现 P0

仅凭一条锚点链接，自动搜索→过滤→评分→输出竞品列表。

**9 阶段流水线**：
```
Stage 0: Anchor Canonicalization
Stage 1: Query Generation → eBay 搜索词
Stage 2: eBay Search → 60 candidates
Stage 3: Light Ranking → title 级打分
Stage 4: Detail Scraping → top 20-25
Stage 5: Canonical Resolution
Stage 6: Hard/Soft Filter → OE Fast Pass
Stage 7: Similarity Scoring → 加权评分
Stage 8: Output → approved / borderline / rejected
Stage 9: DB Persistence → market.db 入库
```

**交付物**：
- `crawler/` — 5 模块（competitor_search / light_ranker / competitor_filter / similarity_engine / database）
- `config/` — 6 配置（vehicle_ontology / query_templates / category_thresholds / filter_rules / similarity_weights / specificity_scores）
- 三项 P0 设计：OE Fast Pass / mismatch vs absent / 复合 key 缓存
- 三项不确定性工程：字段覆盖率降权 / Universal Fit 识别 / 来源特异性覆盖
- Description Source Integration：抓取层新增 description_text 提取（#desc_ifr iframe）；过滤层新增 product_semantics 硬规则（cover/overlay vs replacement 互斥 reject）

## V3（🔜 规划中）：精度优化 + 商业维度

### P1 — 精度优化（跑一轮数据后迭代）

- **Platform/Chassis 平台层级**：识别 GMT800/GMT900 等 GM 平台共享件，跨车型但同平台的候选应加分而非 reject
- **Jaccard 年份重叠**：替代当前简单交集判断，更精细的年份兼容性评分
- **品类差异化权重**：Running Board 的 cab 权重应高于 Fender Flare，不同品类对字段敏感度不同
- **视觉模型参与验证**：GLM-4V 对比 anchor 参考图和 candidate 图片，图≠文时触发告警

### P2 — 商业维度（P1 稳定后）

- **价格/销量纳入排序**：售价相近、销量高的竞品加权更高（更可能是直接竞品）
- **eBay Finding API 替代 Playwright 搜索**：降低反爬风险，提升搜索速度
- **LLM 二轮审核**：DeepSeek 复核 borderline cases（相似度在阈值附近的边缘案例）

## V4（📋 远期）：汽配产品知识库

建立结构化产品类别树，每个类别关联标准规则。

**类别体系**：
| 大类 | 示例产品 | 方位体系 | 特殊规则 |
|------|---------|---------|---------|
| 车门件 | 车窗/把手/铰链/锁/门板/门镜 | Front/Rear = 前门/后门 | 受车门数规则影响 |
| 车体件 | 保险杠/中网/翼子板/机盖/尾门 | Front/Rear = 车头/车尾 | 不受车门数影响 |
| 照明件 | 大灯/尾灯/雾灯/转向灯 | 方位体系独立 | 左右对称规则 |
| 底盘件 | 控制臂/球头/拉杆/减震 | 方位体系独立 | — |
| 动力件 | 进气/排气/冷却/燃油 | 无方位概念 | — |
| 内饰件 | 仪表板/门板/座椅/顶棚 | 方位体系独立 | — |

**每条目包含**：标准方位体系、关键词别名表、竞品匹配规则、eBay Category ID、品类差异化阈值。

格式：JSON Schema，支持 DeepSeek 自动匹配 + 人工校验。

## 执行原则

1. **P0 缺一不可，P1/P2 跑一轮数据后再迭代**，不提前做假设性优化
2. **每个 Phase 至少验证 3 个 SKU 再进下一 Phase**
3. **配置驱动 > 硬编码**，所有规则/阈值/权重进 JSON
4. **mismatch ≠ absent**，未提及 ≠ 冲突
